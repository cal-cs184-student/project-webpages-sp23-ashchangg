Overview

In this project, we implemented the following:

  • Bezier curves and surfaces using de Casteljau’s algorithm which allows us to model smooth and infinitely scalable curves and surfaces
  • Triangle mesh manipulation (edge flips, edge splits) using the half-edge data structure to implement commonly used geometric operations
  • Loop subdivision which allows us to convert coarse polygon meshes into higher-resolution meshes

Overall, we learned a lot about different methods used to improve computer graphics (smooth shading, loop subdivision, etc.) and how to implement these schemes.

Part 2: Bezier Surfaces with Separable 1D de Casteljau

de Casteljau’s algorithm extends to Bezier surfaces as well. We can consider the n x n grid of control points as n different Bezier curves, one for each row, paramaterized by the parameter u. We can evaluate each of these with our previous algorithm to create one point per row, and then we have n points, which can be used themselves to form another Bezier curve with the parameter v. We can now evaluate this with our algorithm from part 1 to create one final point at the parameter v, which is on the Bezier surface.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

In this part, we implemented the area-weighted vertex normals by iterating through all neighboring faces in respect to the vertex (i.e. faces that contain the given vertex as one of its vertices). We then sum up the area-weighted normals of each face, and normalize this value. We then can use these vertex normals for Phong shading which provides better shading for smooth surfaces such as the teapot below.

Part 5: Edge Split

We implemented the edge split operation by first drawing out the locations of all of the half-edges, vertices, edges, and faces before the split similar to the previous task. Then, we create a new vertex, setting its position to the midpoint of the original edge, and create all other additional elements created by this split. Finally, we update all of the existing mesh elements with their new properties after the split.

Part 6: Loop Subdivision for Mesh Upsampling

The loop subdivision algorithm was the most difficult part of the project. Our implementation followed the following steps:

  • First, we computed the new positions for both the new and the old vertices, using the algorithms written in the assignment spec. For the new vertices, we stored the new positions in the corresponding edge, as the vertices did not exist yet.
  • Then, we went through all of the possible edge splits. We make sure to split every edge. As we split, we mark each edge that we create as a new edge, and if we reach an edge that we have already split, we are careful to not split that one (as it would create an infinite loop.) We decided to not mark the edges that was part of the original edge pre-split as new edges, so we had to check to see if an edge was connecting a new vertex with an old vertex (in which case we did not split it further).
  • After this, we iterated through all of the edges, and flipped the edges that were both new, and connected old vertices with new vertices.
  • Finally, we replaced all of the old vertices’ positions with their new position, and then iterated through the edges and placed the new vertex on them at their new positions. We also reset all of the edges and vertices to no longer be classified as “new” for future subdivisions.

When debugging, one difficult problem that we ran into was that some of the vertices were at the wrong positions. To debug this, we printed out all of the values that went into the algorithms to calculate new positions, and found that our “A” and “D” vertices were actually the same vertices, which allowed us to identify and fix the problem.
